# YouTube Watch Tracker - 종합

## 개요

유튜브 영상 시청이 1회성 소비로 흘러가는 것에 대한 아쉬움에서 출발한 프로젝트. "내가 뭘 봤지?"를 기록하고, 기억을 되살리며, 시간 사용을 객관적으로 인지할 수 있는 도구를 만들고자 함.

## 핵심 인사이트

1. **목표는 "컨트롤"이 아닌 "인지"** — 영상 시청 시간을 줄이려는 게 아니라, 내가 뭘 하고 있는지 선명하게 보고 싶음

2. **두 가지 감정적 가치**
   - 기억 되살리기: "아맞아, 이런 내용 있었지" — 불빛이 반짝이는 듯한 느낌
   - 솔직한 회고: "의미없는 영상 많이 봤구나" — 자기 성찰

3. **지식 콘텐츠에 대한 암묵적 기대** — 경제지식, 저축, 신제품 등 "알고 있으면 좋은" 정보를 소비하는데, 기억에 남지 않으면 아깝다고 느낌

4. **Push가 메인, Pull은 보조** — 능동적으로 대시보드를 열어보진 않을 것. 매거진처럼 배달되는 이메일이 주 접점

5. **시청 패턴** — 아침 출근 준비, 저녁 식사 후/씻고 나서. 하루의 시작과 끝에 위치한 루틴

## 제약사항 및 경계

- 나 혼자 쓰고 나만 볼 것 → 프라이버시 걱정 없음
- 모든 영상 정보를 가져와야 함 (필터링 X, 전체 O)
- 부끄럽거나 의미없는 영상도 포함 → 솔직한 회고 가능해야 함

## 결정사항

- **주기**: 주간 이메일, 매주 월요일 오전
- **이메일 내용**: 요약 없이, 대시보드 링크만 포함
- **대시보드**: 상시 접근 가능, 같은 데이터를 언제든 볼 수 있음
- **뷰 구조**:
  - 타임라인 뷰: 날짜/시간순 나열, 각 영상에 카테고리 태그
  - 카테고리 뷰: 주제별로 그룹핑
- **카테고리**: YouTube 기본 카테고리가 아닌, LLM이 세밀하게 분류 (경제지식, 신제품리뷰, 저축팁 등)
- **필수 데이터**: 시청 일시, 영상 정보(제목/썸네일/설명), 시청 시간
- **Optional**: 영상 요약 (LLM 활용)

## 접근방식 및 방향

개인용 유튜브 시청 기록 트래커:
1. YouTube API (또는 브라우저 자동화)로 시청 기록 수집
2. LLM으로 영상별 카테고리 분류 + 선택적 요약
3. 대시보드에서 타임라인/카테고리 뷰 제공
4. 주간 이메일로 대시보드 링크 발송

## 보류된 항목

- 영상 댓글 분석 (투머치로 판단)
- 영상 transcription 기반 요약 (optional로 분류, 우선순위 낮음)
- 인사이트 직접 작성 기능 (있으면 좋지만 optional)
