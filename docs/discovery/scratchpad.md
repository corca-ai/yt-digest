# Interview Scratchpad / 인터뷰 메모 - YouTube Watch Tracker

## Context / 맥락
- Date / 날짜: 2026-04-16
- Reference / 참고자료: 사용자 초기 아이디어 메모

## 초기 입력 (사용자가 정리한 내용)
- **핵심 문제**: 유튜브 영상 시청이 1회성 소비로 흘러감. "내가 뭘 봤지?"가 기록되면 덜 아깝게 느껴질 것
- **입력 데이터 구상**:
  - 시청 일시, 영상 정보, 시청 시간
  - 영상 내용: 제목/썸네일/설명 (필수), 요약 (optional), 댓글 (투머치)
- **처리**: YouTube API 또는 브라우저 자동화
- **출력**: 영상별 데이터 + 일/주 단위 요약 및 인사이트
- **강조점**: "몇월 몇일 무슨 요일 아침에 무슨 내용의 영상을 봤구나" - 가공된 형태가 중요

## Emerging Themes / 주요 주제
(Update as patterns surface / 패턴이 나타나면 업데이트)
- 시청 타이밍: 아침 출근 준비 + 저녁 (식사 후, 씻고 나서) → 하루의 시작과 끝에 위치
- 일상 루틴에 녹아있는 소비 패턴
- 콘텐츠 유형: 경제지식, 저축, 신제품 소개 → "알고 있으면 좋은" 실용 정보 위주
- 삶의 의사결정에 도움되는 정보를 소비하고 있음 (재테크, 소비 선택)

## Key Insights / 핵심 인사이트
(Important points stated by the interviewee / 인터뷰 대상자가 밝힌 중요한 사항)
- "시간을 어떻게 쓰나 생각해봤는데, 생각보다 영상을 많이 보는 것 같았어. 근데 그 영상들의 내용이 기억이 안 나더라고."
- 트리거: 시간 사용에 대한 자기 성찰 → 영상 소비량 인지 → 내용 기억 안 남 → 아까움
- 목표: "컨트롤까진 아니어도 정확히 인지" → 행동 변화가 아닌 자기 인식이 핵심
- 시간을 줄이려는 게 아니라, 내가 뭘 하고 있는지 객관적으로 보고 싶음

## Constraints & Boundaries / 제약사항 및 경계
(Non-negotiables, limitations, boundaries / 필수조건, 제한사항, 경계)
- 나 혼자 쓰고 나만 볼 것 → 프라이버시 걱정 없음, 필터링 불필요
- 모든 영상 정보를 가져와야 함 (선별 X, 전체 O)
- 부끄러운 영상, 의미없는 영상도 포함 → 솔직한 회고 가능
- 필수 데이터: 시청 일시, 영상 정보(제목/썸네일/설명), 시청 시간
- Optional: 영상 요약 (LLM 활용)

## Decisions Made / 결정사항
(Things already decided / 이미 결정된 사항)
- 형태: 하이브리드
  - 핵심 데이터: 시청 기록 + 영상 정보 + 요약 (항상 존재)
  - 뷰 1 (push): 주간 이메일 (월요일 오전) - 일주일치를 묶어서 보여줌
  - 뷰 2 (pull): 대시보드 - 같은 데이터를 상시 접근 가능
- 리포트는 데이터의 "주간 스냅샷"이고, 대시보드는 "실시간 접근"
- 이메일 = 내용 X, 링크만 O → "이번 주 리포트 보러가기" 정도면 충분

## Tensions / Trade-offs / 긴장관계 및 트레이드오프
(Conflicting goals or competing priorities / 상충하는 목표나 우선순위)
- 지식전달 영상을 많이 봄 → 기억에 남으면 "알고 있는 게 많아지고 도움이 될 것" → 그런데 어떤 도움인지는 명확하지 않음
- 암묵적 기대: 지식 콘텐츠 소비 = 나에게 축적되어야 함. 하지만 실제로는 휘발됨
- Push > Pull: 매거진처럼 배달받는 형태가 맞음. 능동적으로 열어보진 않을 것
