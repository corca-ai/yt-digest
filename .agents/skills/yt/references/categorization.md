# Categorization Guide

카테고리 분류와 한 줄 요약은 스킬(Claude)이 직접 수행한다. 키워드 매처는 제거되었다.

## 입력

각 영상은 최소 이 정보를 갖는다.

- `title` — 영상 제목
- `channel` — 채널명
- `channelId` — `data/channels/<channelId>.json` 캐시 키
- `channelUrl` — 채널 URL
- `duration`, `isShort`, `progressPercent` — 보조 신호
- `data/channels/<channelId>.json`이 있으면:
  - `description` — 채널 소개 (meta description 기반, ~160자)
  - `recentVideoTitles` — 최근 영상 제목 최대 20개

## 판정 순서

1. **채널 캐시 우선**: `recentVideoTitles`의 경향성과 `description`이 가장 강한 신호다. 한 채널은 대부분 하나의 카테고리에 쏠린다.
2. **제목으로 보정**: 같은 채널이라도 개별 영상이 다른 주제면 제목이 이긴다.
3. **애매하면 `기타`**: 신호가 충돌하거나 너무 약하면 억지로 분류하지 않는다.

## 카테고리별 시그널

### 기술/개발

- 개발/엔지니어링 채널, 프로그래밍 튜토리얼, AI/LLM, 디자인 시스템, 도구 리뷰
- 예: "Claude Code", "피그마 디자인 시스템", "React 상태관리", "GPT 파인튜닝"

### 비즈니스/경제

- 창업/경영/투자/시장/경제 지표, 스타트업 인터뷰, 경영진 팟캐스트
- 예: "노정석" 같은 스타트업 팟캐스트, "에픽테토스의 CEO", 투자 리포트

### 교육/자기계발

- 강의, 북리뷰, 생산성, 학습법, 인사이트 강연
- 기술 강의라도 초점이 "배움/정리"면 여기. 특정 스택의 튜토리얼은 기술/개발.

### 엔터테인먼트

- 예능/쇼/게임/리액션/영화 리뷰/코미디

### 음악/예배

- 음악, 연주, 커버, 콘서트, 찬양/예배/CCM

### 뷰티/패션

- 메이크업, 스타일링, 하울, 스킨케어

### 먹방/요리

- 레시피, 맛집, 먹방

### 브이로그/일상

- 일상, 여행, 루틴, 브이로그

### 뉴스/시사

- 속보, 정치, 사회 이슈, 토론

### 기타

- 위 어디에도 속하지 않거나 신호가 상충/불명확

## 요약 작성

- 한국어 1문장, 60자 내외
- "무엇을 다루는 영상인지"를 담고, 채널/제목을 그대로 복사하지 않는다
- Shorts는 `"[Shorts] ..."` 접두사 권장

## 출력 규약

- 같은 `data/history/history-YYYY-MM-DD.json`을 **부분 업데이트**한다 (다른 필드 보존)
- `category`는 `references/categories.md`의 10개 값 중 하나
- `summary`는 문자열 (빈 문자열 금지; 정보가 없으면 제목 기반 fallback)
- JSON 포맷: 2 space indent, 끝 개행 유지
